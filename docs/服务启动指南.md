# AvatarAgent 服务启动指南

## 服务总览

| 服务 | 端口 | 说明 |
|------|------|------|
| 后端 API | 8000 | FastAPI 后端服务 |
| 前端 | 5173 | Vue 3 + Vite 开发服务器 |
| CosyVoice TTS | 50000 | 语音合成服务 |
| HeyGem 数字人 | 8383 | 数字人视频生成 |
| TuiliONNX 数字人 | 8384 | 数字人视频生成（备选） |
| Whisper ASR | 本地 | 语音识别（按需自动加载） |

---

## 1. 后端服务

后端基于 FastAPI + Uvicorn，入口为项目根目录的 `launcher.py`。

### 启动

```bash
# 激活虚拟环境
.venv\Scripts\activate        # Windows
# source .venv/bin/activate   # Linux/Mac

# 启动后端
python launcher.py
```

启动后：
- API 服务地址：`http://localhost:8000`
- API 文档（Swagger）：`http://localhost:8000/docs`

### 配置

编辑 `config/config.ini`，主要配置项：

```ini
[server]
host = 0.0.0.0
port = 8000

[ffmpeg]
bin_path = /path/to/ffmpeg.exe

[deepseek_apikey]
key = your_api_key
```

## 2. 前端服务

前端基于 Vue 3 + TypeScript + Vite。

### 启动

```bash
cd frontend

# 首次运行需安装依赖
npm install

# 启动开发服务器
npm run dev
```

启动后访问：`http://localhost:5173`

### 其他命令

```bash
npm run build      # 构建生产版本
npm run preview    # 预览生产构建
```

## 3. CosyVoice 语音合成服务

CosyVoice 提供 TTS（文字转语音）能力，**必须使用其独立虚拟环境** `third_party/CosyVoice/.venv` 启动，不能用主项目 Python，否则缺少 `wetext` 等依赖，导致文本前端加载失败（HTTP 500）。

### 推荐启动方式（通过后端 API 触发，自动使用正确 Python）

在后端服务启动后，访问设置页面 → "CosyVoice 运行设置" → 点击"重启服务"即可。
后端会自动使用 `third_party/CosyVoice/.venv/Scripts/python.exe` 启动。

### 手动启动（Windows）

> ⚠️ **必须使用 `.venv` 里的 Python，禁止用主 Python 直接启动**

```bat
REM 设置必要的环境变量（禁止联网，清除系统代理）
set PYTHONPATH=E:\Software\VideoAgent\AvatarAgent\third_party\CosyVoice
set MODELSCOPE_CACHE=E:\Software\VideoAgent\AvatarAgent\third_party\models\modelscope_cache
set MODELSCOPE_OFFLINE=1
set NO_PROXY=*
set HTTP_PROXY=
set HTTPS_PROXY=
set http_proxy=
set https_proxy=
set ALL_PROXY=
set all_proxy=

REM 用 .venv 里的 Python 启动（关键！）
E:\Software\VideoAgent\AvatarAgent\third_party\CosyVoice\.venv\Scripts\python.exe ^
  E:\Software\VideoAgent\AvatarAgent\third_party\CosyVoice\runtime\python\fastapi\server.py ^
  --port 50000 ^
  --model_dir E:\Software\VideoAgent\AvatarAgent\third_party\models\modelscope_cache\hub\iic\CosyVoice2-0___5B ^
  --device gpu
```

### 首次安装（如 `.venv` 不存在）

```bash
# 在项目根目录执行（使用主项目 .venv）
python -m src.audio.cosyvoice_deploy setup
```

### 健康检查

```bash
python -m src.audio.cosyvoice_deploy check --port 50000
```

### 配置

```ini
[cosyvoice]
host = 127.0.0.1
port = 50000
sample_rate = 0    # 0 = 自动检测（CosyVoice2 为 24000Hz）
device = gpu
model_dir =        # 留空则自动使用 third_party/models/... 下的本地缓存
```

### 注意事项

- **必须用** `third_party/CosyVoice/.venv/Scripts/python.exe` 启动，主项目 Python 缺少 `wetext`，会导致文本前端加载失败（日志显示 `no frontend is avaliable`，合成时返回 HTTP 500）
- 启动时必须设置 `MODELSCOPE_OFFLINE=1` 并清除所有代理环境变量（`HTTP_PROXY`/`HTTPS_PROXY`/`ALL_PROXY`）。wetext 内部的 `snapshot_download` 不遵守该标志，项目已在 `.venv/Lib/site-packages/sitecustomize.py` 中安装了自动 patch，配合 `MODELSCOPE_OFFLINE=1` 实现真正离线
- 模型缓存位于 `third_party/models/modelscope_cache/`，已预先下载，完全离线运行
- 如需更新模型，在前端设置页面点击"更新模型"按钮手动触发（该操作会临时联网）

## 5. Whisper 语音识别（ASR）

Whisper 用于视频/音频转文字，支持三种模式：

### 模式 A：本地 GPU（默认）

无需单独启动，首次调用时自动加载模型。

```ini
[whisper]
model_size = base    # 可选: turbo, large, medium, base, small, tiny
model_dir = /path/to/whisper/models
```

### 模式 B：本地 CPU

GPU 不可用时自动回退，无需额外配置。

### 模式 C：云端 GPU（AutoDL 等远程服务器）

在远程服务器上部署 Whisper API：

```bash
# 远程服务器
pip install openai-whisper fastapi uvicorn python-multipart
python docs/autodl_whisper_server.py
# 默认端口 6006
```

本地配置：

```ini
[cloud_gpu]
enabled = true
api_url = http://remote-server:6006
api_key = your_key    # 可选
```

## 7. 推荐启动顺序

1. 启动 CosyVoice TTS 服务（如需语音合成）
2. 启动 HeyGem / TuiliONNX 数字人服务（如需数字人视频）

5. 浏览器访问 `http://localhost:5173`

## 8. 常见问题

**Q: 后端启动报错找不到 FFmpeg？**
在 `config/config.ini` 的 `[ffmpeg]` 中配置 `bin_path` 指向 ffmpeg 可执行文件路径。

**Q: CosyVoice 启动失败？**
检查 `third_party/CosyVoice/.venv` 是否存在，若不存在需先创建独立虚拟环境并安装依赖。

**Q: 前端无法连接后端？**
确认后端已启动且运行在 8000 端口，前端 Vite 开发服务器已配置代理到后端地址。
