# AvatarAgent 服务启动指南

## 服务总览

| 服务 | 端口 | 说明 |
|------|------|------|
| 后端 API | 8000 | FastAPI 后端服务 |
| 前端 | 5173 | Vue 3 + Vite 开发服务器 |
| CosyVoice TTS | 50000 | 语音合成服务 |
| HeyGem 数字人 | 8383 | 数字人视频生成 |
| TuiliONNX 数字人 | 8384 | 数字人视频生成（备选） |
| Whisper ASR | 本地 | 语音识别（按需自动加载） |

---

## 1. 后端服务

后端基于 FastAPI + Uvicorn，入口为项目根目录的 `launcher.py`。

### 启动

```bash
# 激活虚拟环境
.venv\Scripts\activate        # Windows
# source .venv/bin/activate   # Linux/Mac

# 启动后端
python launcher.py
```

启动后：
- API 服务地址：`http://localhost:8000`
- API 文档（Swagger）：`http://localhost:8000/docs`

### 配置

编辑 `config/config.ini`，主要配置项：

```ini
[server]
host = 0.0.0.0
port = 8000

[ffmpeg]
bin_path = /path/to/ffmpeg.exe

[deepseek_apikey]
key = your_api_key
```

## 2. 前端服务

## 3. CosyVoice 语音合成服务

CosyVoice 提供 TTS（文字转语音）能力，**必须使用其独立虚拟环境** `third_party/CosyVoice/.venv` 启动，不能用主项目 Python，否则缺少 `wetext` 等依赖，导致文本前端加载失败（HTTP 500）。

### 推荐启动方式（通过后端 API 触发，自动使用正确 Python）

在后端服务启动后，访问设置页面 → "CosyVoice 运行设置" → 点击"重启服务"即可。
后端会自动使用 `third_party/CosyVoice/.venv/Scripts/python.exe` 启动，并将日志写入 `logs/cosyvoice.log`。

> ⚠️ **不要用命令行手动后台启动**：若通过 `subprocess` 或 `start /min` 以 PIPE 方式捕获 stdout/stderr 启动，`tqdm` 会在推理时尝试 flush PIPE 导致 `OSError: [Errno 22] Invalid argument`，所有合成请求返回 HTTP 500。务必让 stdout/stderr 写入真实文件（后端 restart API 已处理此问题）。

### 手动启动（Windows）

> ⚠️ **必须使用 `.venv` 里的 Python，禁止用主 Python 直接启动**

```bat
REM 设置必要的环境变量（禁止联网，清除系统代理）
set PYTHONPATH=E:\Software\VideoAgent\AvatarAgent\third_party\CosyVoice
set MODELSCOPE_CACHE=E:\Software\VideoAgent\AvatarAgent\third_party\models\modelscope_cache
set MODELSCOPE_OFFLINE=1
set NO_PROXY=*
set HTTP_PROXY=
set HTTPS_PROXY=
set http_proxy=
set https_proxy=
set ALL_PROXY=
set all_proxy=

REM 用 .venv 里的 Python 启动（关键！）
E:\Software\VideoAgent\AvatarAgent\third_party\CosyVoice\.venv\Scripts\python.exe ^
  E:\Software\VideoAgent\AvatarAgent\third_party\CosyVoice\runtime\python\fastapi\server.py ^
  --port 50000 ^
  --model_dir E:\Software\VideoAgent\AvatarAgent\third_party\models\modelscope_cache\hub\iic\CosyVoice2-0___5B ^
  --device gpu
```

### 首次安装（如 `.venv` 不存在）

```bash
# 在项目根目录执行（使用主项目 .venv）
python -m src.audio.cosyvoice_deploy setup
```

### 健康检查

```bash
python -m src.audio.cosyvoice_deploy check --port 50000
```

### 配置

```ini
[cosyvoice]
host = 127.0.0.1
port = 50000
sample_rate = 0    # 0 = 自动检测（CosyVoice2 为 24000Hz）
device = gpu
model_dir =        # 留空则自动使用 third_party/models/... 下的本地缓存
```

### 注意事项

- **必须用** `third_party/CosyVoice/.venv/Scripts/python.exe` 启动，主项目 Python 缺少 `wetext`，会导致文本前端加载失败（日志显示 `no frontend is avaliable`，合成时返回 HTTP 500）
- 启动时必须设置 `MODELSCOPE_OFFLINE=1` 并清除所有代理环境变量（`HTTP_PROXY`/`HTTPS_PROXY`/`ALL_PROXY`）。wetext 内部的 `snapshot_download` 不遵守该标志，项目已在 `.venv/Lib/site-packages/sitecustomize.py` 中安装了自动 patch，配合 `MODELSCOPE_OFFLINE=1` 实现真正离线
- 模型缓存位于 `third_party/models/modelscope_cache/`，已预先下载，完全离线运行
- 如需更新模型，在前端设置页面点击"更新模型"按钮手动触发（该操作会临时联网）
- **stdout/stderr 不能是 PIPE**：以 PIPE 方式启动会导致 `tqdm` 推理时 `OSError: [Errno 22]`，所有合成返回 HTTP 500。后端 restart API 已将日志重定向到 `logs/cosyvoice.log` 解决此问题
- **日志路径**：`logs/cosyvoice.log`（通过后端 restart API 启动时自动创建）

## 4. HeyGem 数字人服务

HeyGem 通过 Docker 容器运行，镜像为 `guiji2025/heygem.ai:latest`，**必须启用 GPU（`--gpus all`）**，否则任务会卡在 `status=1` 长达数分钟后失败。

### 首次启动（创建容器）

```bat
docker run -d --name heygem --gpus all ^
  -p 8383:8383 ^
  -v "E:\Software\VideoAgent\AvatarAgent\output:/heygem_data/voice/data" ^
  -v "E:\Software\VideoAgent\AvatarAgent\resources\models:/heygem_data/face2face" ^
  -v "E:\Software\VideoAgent\AvatarAgent\output\avatar:/code/data/temp" ^
  -w /code ^
  --entrypoint /usr/local/python3/bin/python3 ^
  guiji2025/heygem.ai:latest app_local.py
```

> ⚠️ **关键注意事项：**
> - 必须加 `--gpus all`，否则 CUDA 初始化失败，任务永远不会完成
> - 启动脚本必须是 `app_local.py`（不是 `app.py`），`app.py` 有 import 错误
> - `-w /code` 和 `--entrypoint /usr/local/python3/bin/python3` 缺一不可
> - 三条 `-v` 挂载缺一不可：`output`=音频输入，`resources/models`=模型视频，`output\avatar`=生成结果输出

### 日常启动（容器已存在）

```bat
docker start heygem
```

或在 Docker Desktop 的 Containers 列表中点击 ▶ 启动。

### 启动成功的标志

日志中出现以下内容即表示服务就绪：

```
* Running on http://127.0.0.1:8383
av_transfer load success
```

### 路径映射配置

```ini
[heygem]
host = 127.0.0.1
port = 8383
audio_host_dir = E:\Software\VideoAgent\AvatarAgent\output
audio_container_dir = /heygem_data/voice/data
video_host_dir = E:\Software\VideoAgent\AvatarAgent\resources\models
video_container_dir = /heygem_data/face2face
```

### 注意事项

- **系统代理问题**：如果电脑开着翻墙代理，httpx 会走代理导致连不上本地服务（报 `All connection attempts failed`）。代码已通过 `transport=httpx.AsyncHTTPTransport()` 绕过，无需额外操作
- **GPU 显存**：RTX 3050 Laptop（4GB 显存）可以运行，模型加载约需 20-30 秒
- **任务卡住不动**：多半是没加 `--gpus all`，需删除容器重新用上面命令创建

### 删除并重建容器

```bat
docker rm -f heygem
REM 然后重新执行「首次启动」命令
```

## 5. Whisper 语音识别（ASR）

Whisper 用于视频/音频转文字，支持三种模式：

### 模式 A：本地 GPU（默认）

无需单独启动，首次调用时自动加载模型。

```ini
[whisper]
model_size = base    # 可选: turbo, large, medium, base, small, tiny
model_dir = /path/to/whisper/models
```

### 模式 B：本地 CPU

GPU 不可用时自动回退，无需额外配置。

### 模式 C：云端 GPU（AutoDL 等远程服务器）

在远程服务器上部署 Whisper API：

```bash
# 远程服务器
pip install openai-whisper fastapi uvicorn python-multipart
python docs/autodl_whisper_server.py
# 默认端口 6006
```

本地配置：

```ini
[cloud_gpu]
enabled = true
api_url = http://remote-server:6006
api_key = your_key    # 可选
```

## 7. 推荐启动顺序

1. **HeyGem**（Docker）：`docker start heygem`
2. **后端**：`python launcher.py`（先于 CosyVoice 启动，因为 CosyVoice 通过后端 API 触发）
3. **CosyVoice TTS**：后端启动后，在设置页面点击"重启服务"（见第 3 节）
4. **前端**：`cd frontend && npm run dev`
5. 浏览器访问 `http://localhost:5173`

> - HeyGem 模型加载需约 20-30 秒，建议最先启动。
> - CosyVoice 模型加载需约 60-90 秒（GPU），等待 `logs/cosyvoice.log` 出现 `Application startup complete` 后再使用。
> - **不要在后端启动前通过命令行手动启动 CosyVoice**，否则 stdout/stderr 为 PIPE 导致推理 500 报错。

## 8. 常见问题

**Q: 后端启动报错找不到 FFmpeg？**
在 `config/config.ini` 的 `[ffmpeg]` 中配置 `bin_path` 指向 ffmpeg 可执行文件路径。

**Q: CosyVoice 启动失败？**
检查 `third_party/CosyVoice/.venv` 是否存在，若不存在需先创建独立虚拟环境并安装依赖。通过后端 restart API 启动，日志在 `logs/cosyvoice.log`。

**Q: CosyVoice 合成一直返回 HTTP 500（`OSError: [Errno 22] Invalid argument`）？**
CosyVoice 进程的 stdout/stderr 被 PIPE 捕获（如通过 `start /min` 或 `subprocess.PIPE` 手动启动），导致 `tqdm` 推理时 flush 报错。解决方法：停止当前 CosyVoice 进程，通过后端设置页面"重启服务"重新启动（后端会将输出重定向到 `logs/cosyvoice.log` 文件）。

**Q: 前端无法连接后端？**
确认后端已启动且运行在 8000 端口，前端 Vite 开发服务器已配置代理到后端地址。

**Q: HeyGem 任务一直 status=1，几分钟后失败？**
容器启动时没有加 `--gpus all`，CUDA 初始化失败。删除容器后用第 4 节的完整命令（含 `--gpus all`）重建。

**Q: HeyGem 报 `All connection attempts failed`？**
电脑开着代理（翻墙软件）时 httpx 会走代理。代码已修复（`transport=httpx.AsyncHTTPTransport()`），若仍报错请确认代码版本是最新的。

**Q: HeyGem 容器名冲突（`name "/heygem" is already in use`）？**
执行 `docker rm heygem` 删除旧容器后重新创建。

**Q: 预览页面视频时长显示为 0，无法播放？**
HeyGem/TuiliONNX 生成的 MP4 文件 `moov` atom 在文件末尾（非 faststart 格式），浏览器无法直接获取时长元数据。代码已在数字人生成完成后自动用 ffmpeg `-movflags +faststart` 重封装（moov 移到文件头）。若仍有问题，确认 `config/config.ini` 中 `[ffmpeg] bin_path` 指向有效的 ffmpeg 可执行文件。
